# 시트 쓰기용 Apps Script 웹앱 배포

HOME 화면의 **새 프로젝트 입력** 폼이 값을 저장하려면, 대상 스프레드시트에
연결된 Apps Script 웹앱을 한 번 배포하고 그 URL을 `.env`의 `SHEET_WEBAPP_URL`에
넣어야 합니다. (공개 시트는 "읽기"만 인증 없이 가능하고, "쓰기"는 인증이 필요하기 때문)

> 이 작업은 **시트를 소유(편집 권한)** 한 계정으로 해야 합니다.

## 1. 스크립트 붙여넣기

1. 대상 구글 시트를 연다. (`src/sheet.ts`의 `SPREADSHEET_ID`가 가리키는 시트)
2. 메뉴 **확장 프로그램 → Apps Script** 클릭.
3. 기본 `Code.gs` 내용을 지우고, 이 폴더의 [`Code.gs`](./Code.gs) 내용을 그대로 붙여넣고 저장(💾).

## 2. 웹앱으로 배포

1. 우측 상단 **배포 → 새 배포**.
2. 톱니바퀴(유형 선택) → **웹 앱**.
3. 설정:
   - **실행 계정(Execute as):** 나(Me)
   - **액세스 권한(Who has access):** **모든 사용자(Anyone)**
     - 로컬 서버가 인증 없이 POST 할 수 있어야 하기 때문. URL은 비밀로 취급되어 `.env`에만 보관됩니다.
4. **배포** → 최초 1회 **권한 승인**(본인 구글 계정) 진행.
5. 표시되는 **웹 앱 URL** 복사 (`https://script.google.com/macros/s/.../exec` 형태).

## 3. .env 에 등록

```bash
SHEET_WEBAPP_URL=https://script.google.com/macros/s/XXXXXXXX/exec
```

저장 후 **API 서버를 재시작**(`npm run server` 또는 `npm run dev:all`)하세요.

## 확인

- 브라우저로 웹앱 URL을 그냥 열면 `{"ok":true,...}` 가 보이면 정상(doGet).
- HOME 폼에서 프로젝트를 입력하고 **입력 완료**를 누르면 시트 맨 아래에 행이 쌓입니다.

## 참고

- 목록(좌측 메뉴)은 구글의 gviz 캐시 때문에 저장 직후 최대 1~2분 갱신이 지연될 수 있습니다.
- 스크립트를 수정하면 **새 버전으로 다시 배포**해야 반영됩니다(배포 → 배포 관리 → 편집 → 새 버전).
